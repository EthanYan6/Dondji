# MDC1200 调试工具使用指南

## 📋 工具简介

这是一个用于调试 MDC1200 信号的 Python 工具，可以帮助你:
- ✅ 录制和分析音频信号
- ✅ 测量频偏 (频率偏差)
- ✅ 频谱分析
- ✅ 波形可视化
- ✅ 生成详细分析报告

## 🔧 环境要求

- Python 3.8 或更高版本
- Windows 7/10/11
- 音频输入设备 (麦克风、声卡等)

## 📦 安装方法

### 方法一：使用打包好的 EXE (推荐)

1. 运行 `build_debugger.bat`
2. 等待打包完成
3. 在 `dist` 目录找到 `MDC1200 调试工具.exe`
4. 双击运行即可

### 方法二：直接运行 Python 脚本

```bash
# 安装依赖
pip install numpy matplotlib scipy sounddevice

# 运行工具
python mdc1200_debugger.py
```

## 🎯 使用场景

### 场景 1: 测量发射频偏

**目的**: 检查你的对讲机发射的 MDC1200 信号频偏是否正确

**步骤**:
1. 准备另一台对讲机或收音设备
2. 将对讲机的耳机输出连接到电脑麦克风输入
   - 或者用另一台对讲机接收，耳机输出到电脑
3. 运行调试工具，点击"开始录制"
4. 在另一台对讲机上按 PTT 发射 MDC1200 信号
5. 录制完成后，点击"频偏测量"

**结果分析**:
- ✅ 正常：频偏误差 < 500Hz (标准值 3500Hz)
- ⚠️ 略有偏差：500Hz < 误差 < 1000Hz，建议调整
- ✗ 异常：误差 > 1000Hz，需要修改代码中的 REG_40 设置

### 场景 2: 分析接收问题

**目的**: 检查为什么接收不到其他机器的 MDC1200 信号

**步骤**:
1. 用标准设备发射 MDC1200 信号
2. 用调试工具录制接收到的音频
3. 点击"频谱分析"查看频率成分
4. 检查是否有 4700Hz (Mark) 和 2300Hz (Space) 分量

**可能的问题**:
- 没有检测到音调 → 信号太弱或频率完全错误
- 只有单一频率 → 可能是单音而不是 FSK
- 频率偏移过大 → 晶振或调制问题

### 场景 3: 调试硬件适配

**目的**: 找到适合你硬件的最佳参数

**需要调整的参数** (在 `bk4829.c` 中):

```c
// 1. 频偏设置 (REG_40)
uint16_t deviation = 3500;  // 标准值，可根据实测调整

// 2. 接收带宽 (REG_58)
(7u << 10)  // 当前值 7，可尝试 5-9

// 3. 接收增益 (REG_58)
(3u << 8)   // 当前值 3，可尝试 0-7
```

**调试流程**:
1. 修改一个参数
2. 重新编译固件
3. 用调试工具测试
4. 记录结果
5. 重复直到找到最佳值

## 📊 输出说明

### 频谱分析
显示信号中的主要频率成分，帮助识别:
- Mark 频率 (应为 ~4700Hz)
- Space 频率 (应为 ~2300Hz)
- 干扰信号

### 频偏测量
显示关键参数:
- 中心频率：载波中心位置
- 最大/最小频率：频率摆动的极值
- 实测频偏：(最大 - 最小) / 2
- 频偏误差：实测值与标准值的差异

### 波形图
三个视图:
1. **时域波形**: 原始音频信号
2. **频谱图**: 频率分布
3. **语谱图**: 频率随时间变化

## 🔍 故障排查

### 问题 1: 无法录音
**原因**: 
- 没有选择正确的音频输入设备
- 设备被其他程序占用

**解决**:
1. 点击"刷新设备"
2. 选择正确的麦克风/线路输入
3. 关闭其他可能占用麦克风的程序

### 问题 2: 频偏测量失败
**原因**:
- 信号太弱
- 录音质量差
- 不是 FSK 信号

**解决**:
1. 增加录音音量/增益
2. 靠近声源或使用直接音频连接
3. 检查发射设备是否正确发送 MDC1200

### 问题 3: 检测不到 Mark/Space 频率
**可能原因**:
- 频偏设置错误
- 音频路径问题
- 硬件限制

**解决**:
1. 检查发射端的频偏设置
2. 尝试调整 REG_40 的值
3. 检查音频耦合电容等硬件

## 💡 调试技巧

### 技巧 1: 对比测试
用已知正常的设备发射，录制后作为参考标准，与待调试设备对比。

### 技巧 2: 逐步调试
1. 先测试单音 (1200Hz) → 确认基本音频路径正常
2. 测试 FSK 交替信号 → 确认调制正常
3. 测试完整 MDC1200 包 → 确认编码正常

### 技巧 3: 记录日志
每次修改参数后，保存分析报告，建立调试日志，方便追踪问题。

## 🛠️ 高级功能

### 修改采样率
在脚本开头修改:
```python
analyzer = MDC1200Analyzer(sample_rate=48000)  # 改为你的声卡采样率
```

### 批量分析
创建批处理脚本分析多个文件:
```python
import glob
for wav_file in glob.glob("*.wav"):
    analyzer.load_wav_file(wav_file)
    report = analyzer.analyze_and_report()
    print(f"\n{wav_file}:\n{report}")
```
