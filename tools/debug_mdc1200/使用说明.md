# MDC1200 调试工具包 - 使用说明

## 📦 工具包内容

本工具包包含以下文件：

### 核心工具
1. **mdc1200_debugger.py** - 主调试程序 (GUI 界面)
2. **quick_test.py** - 快速测试脚本 (命令行)
3. **run_debugger.bat** - 一键启动菜单
4. **build_debugger.bat** - 打包成 EXE 的脚本

### 文档
5. **README_MDC1200_DEBUGGER.md** - 调试工具使用指南
6. **MDC1200_PATCH_GUIDE.md** - 固件修改指南
7. **requirements.txt** - Python 依赖列表

## 🚀 快速开始

### 方法 1: 使用启动菜单 (推荐)

```bash
# 双击运行
run_debugger.bat

# 或在命令行
.\run_debugger.bat
```

然后按照菜单提示选择操作。

### 方法 2: 手动操作

#### 步骤 1: 安装依赖
```bash
pip install -r requirements.txt
```

#### 步骤 2: 运行调试工具
```bash
# GUI 界面
python mdc1200_debugger.py

# 或命令行快速测试
python quick_test.py your_audio.wav
```

#### 步骤 3: 打包成 EXE
```bash
build_debugger.bat
```

打包完成后，EXE 文件在 `dist` 目录。

## 🎯 典型使用场景

### 场景 1: 调试发射频偏

**目标**: 确保发射的 MDC1200 信号符合标准

**步骤**:
1. 准备测试设备
   - 对讲机 → 假负载 + 耦合线圈 → 音频线 → 电脑麦克风输入
   - 或者：对讲机耳机输出 → 音频线 → 电脑麦克风输入

2. 启动调试工具
   ```bash
   .\run_debugger.bat
   选择 [2] 直接运行 Python 调试工具
   ```

3. 在 GUI 中:
   - 选择正确的音频输入设备
   - 设置录制时长 (建议 5 秒)
   - 点击"开始录制"

4. 在对讲机上按 PTT 发射 MDC1200

5. 录制完成后:
   - 点击"频偏测量"
   - 查看结果

**期望结果**:
```
实测频偏：3400-3600 Hz
频偏误差：±100 Hz
评级：优秀
```

**如果结果异常**:
- 参考 `MDC1200_PATCH_GUIDE.md` 修改固件
- 主要修改 `App/driver/bk4829.c` 中的频偏设置

### 场景 2: 分析接收问题

**目标**: 找出为什么接收不到其他机器的信号

**步骤**:
1. 用标准设备发射 MDC1200 信号
2. 用调试工具录制接收到的音频
3. 点击"频谱分析"
4. 检查是否有 4700Hz 和 2300Hz 成分

**可能的问题**:
- ❌ 没有检测到频率 → 信号太弱或设备故障
- ⚠️ 只有单一频率 → 不是 FSK 调制
- ✗ 频率偏差大 → 晶振或调制问题

### 场景 3: 批量测试

**目标**: 测试多个录音文件

**步骤**:
```bash
# 使用快速测试脚本
python quick_test.py test1.wav
python quick_test.py test2.wav
python quick_test.py test3.wav
```

## 📊 结果解读

### 频偏测量结果

| 误差范围 | 评级 | 建议 |
|----------|------|------|
| ±300 Hz | ✓ 优秀 | 无需调整 |
| ±300-800 Hz | ⚠ 良好 | 可以微调 |
| >±800 Hz | ✗ 异常 | 必须修改 |

### 频谱分析结果

**正常 MDC1200 信号**:
- 应该在 4700Hz 附近有峰值 (Mark)
- 应该在 2300Hz 附近有峰值 (Space)
- 两个峰值幅度相近

**异常信号**:
- 只有一个峰值 → 可能是单音
- 峰值位置不对 → 频偏错误
- 峰值很宽 → 频率不稳定

## 🔧 固件修改流程

### 1. 备份原文件
```bash
copy App\driver\bk4829.c App\driver\bk4829.c.bak
```

### 2. 修改频偏
编辑 `App\driver\bk4829.c`，找到 `BK4819_PlayMDC1200` 函数:

```c
// 修改前
uint16_t deviation = 850;

// 修改后
uint16_t deviation = 3500;
```

### 3. 编译固件
```bash
make clean
make
```

### 4. 烧录并测试
- 烧录新固件
- 用调试工具测试
- 对比修改前后的结果

## 💡 调试技巧

### 技巧 1: 对比法
用已知正常的设备发射，录制后作为参考标准。

### 技巧 2: 逐步逼近
1. 先测单音 (1200Hz) → 确认音频路径
2. 再测 FSK → 确认调制
3. 最后测完整包 → 确认编码

### 技巧 3: 记录日志
每次修改都记录:
- 修改内容
- 实测结果
- 通信效果

### 技巧 4: 多设备验证
用多台设备互相测试，确认问题不是个例。

## 🐛 常见问题

### Q1: 运行时提示缺少模块
**解决**: 
```bash
pip install -r requirements.txt
```

### Q2: 无法录音
**解决**:
- 检查麦克风权限
- 选择正确的音频设备
- 关闭其他占用麦克风的程序

### Q3: 打包失败
**解决**:
```bash
pip install --upgrade pyinstaller
```

### Q4: 频偏总是偏差很大
**解决**:
- 检查晶振是否准确
- 检查音频耦合电路
- 可能需要硬件修改
