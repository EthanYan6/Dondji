# MDC1200 固件修改指南

## 🎯 问题分析

当前仓库中的 MDC1200 实现存在问题:
- ❌ 本机发射，其他机器无法解析
- ❌ 其他机器发射，本机无法解析

**根本原因**: 频偏 (Frequency Deviation) 设置不符合 MDC1200 标准

## 📊 MDC1200 标准参数

| 参数 | 标准值 | 当前值 | 状态 |
|------|--------|--------|------|
| 调制方式 | FFSK (Fast FSK) | FFSK | ✓ |
| 波特率 | 1200 baud | 1200 baud | ✓ |
| Mark 频率 | 4700 Hz (1200+3500) | ? | 需验证 |
| Space 频率 | -2300 Hz (1200-3500) | ? | 需验证 |
| **频偏** | **±3.5 kHz** | **~850 Hz** | ✗ **异常** |
| 同步字 | 0x07092A446F | 0x07092A446F | ✓ |

## 🔧 关键修改

### 修改 1: 修正频偏 (最重要)

**文件**: `App\driver\bk4829.c`  
**函数**: `BK4819_PlayMDC1200`  
**位置**: 约 1867 行

```c
// 原代码
uint16_t deviation = 850;
if (gCurrentVfo != NULL) {
    switch (gCurrentVfo->CHANNEL_BANDWIDTH) {
        case BK4819_FILTER_BW_WIDE:     deviation = 1050; break;
        case BK4819_FILTER_BW_NARROW:   deviation = 850; break;
        case BK4819_FILTER_BW_NARROWER: deviation = 750; break;
        default:                        deviation = 850; break;
    }
}

// 修改为 - MDC1200 标准频偏 ±3.5kHz
uint16_t deviation = 3500;  // 固定为 MDC1200 标准值
if (gCurrentVfo != NULL) {
    switch (gCurrentVfo->CHANNEL_BANDWIDTH) {
        case BK4819_FILTER_BW_WIDE:     deviation = 3750; break;  // 略大以补偿带宽
        case BK4819_FILTER_BW_NARROW:   deviation = 3500; break;  // 标准值
        case BK4819_FILTER_BW_NARROWER: deviation = 3250; break;  // 略小
        default:                        deviation = 3500; break;
    }
}
```

**说明**: 
- MDC1200 使用 ±3.5kHz 频偏，这是标准规范
- 原代码的 850Hz 远小于标准值，导致其他设备无法正确解调
- 不同带宽下可以微调，但都应该在 3500Hz 附近

### 修改 2: 优化接收带宽

**文件**: `App\driver\bk4829.c`  
**函数**: `BK4819_EnableMDC1200Rx`  
**位置**: 约 1980 行

```c
// 原代码
BK4819_WriteRegister(BK4819_REG_58,
    (1u << 13) |    // FSK Enable
    (7u << 10) |    // RX Bandwidth = 7
    (3u <<  8) |    // RX Gain = 3
    (0u <<  6) |
    (0u <<  4) |
    (1u <<  1) |
    (1u <<  0));

// 修改为 - 优化接收带宽和增益
BK4819_WriteRegister(BK4819_REG_58,
    (1u << 13) |    // FSK Enable
    (6u << 10) |    // RX Bandwidth = 6 (更窄，提高信噪比)
    (5u <<  8) |    // RX Gain = 5 (提高灵敏度)
    (0u <<  6) |
    (0u <<  4) |
    (1u <<  1) |
    (1u <<  0));
```

**REG_58 位说明**:
- Bit 13: FSK 使能
- Bit 10-12: RX 带宽 (值越小带宽越窄)
- Bit 8-9: RX 增益 (值越大增益越高)

**带宽选择建议**:
- 如果干扰大 → 减小带宽 (5-6)
- 如果信号弱 → 增加带宽 (7-8)
- 标准值：6-7

### 修改 3: 调整 Tone2 增益

**文件**: `App\driver\bk4829.c`  
**函数**: `BK4819_PlayMDC1200`  
**位置**: 约 1888 行

```c
// 原代码
BK4819_WriteRegister(BK4819_REG_70, 0x00E0);  // Tone2 增益 = 0xE0 = 224

// 修改为 - 降低增益避免失真
BK4819_WriteRegister(BK4819_REG_70, 0x00A0);  // Tone2 增益 = 0xA0 = 160
```

**说明**:
- REG_70 的 Bit 6-0 控制 Tone2/FSK 幅度
- 过高的增益会导致音频失真
- 160 (0xA0) 是更合适的值

### 修改 4: 优化音频路径

**文件**: `App\driver\bk4829.c`  
**函数**: `BK4819_PlayMDC1200`  
**位置**: 约 1847 行

```c
// 在函数开头添加
AUDIO_AudioPathOff();  // 关闭音频输出到扬声器

// 确保 REG_50 设置正确
BK4819_WriteRegister(BK4819_REG_50, 0x3B20);  // 保持当前设置
```

**说明**:
- MDC1200 发射时不应该输出到扬声器
- 确保音频路径只到调制器

## 🐛 常见问题

### Q1: 频偏还是不对怎么办？

**A**: 检查以下几点:
1. 确认修改的是正确的文件 (bk4829.c 不是 bk4819.c)
2. 确认编译时使用了新代码 (make clean 后重新编译)
3. 确认烧录的是新固件
4. 用调试工具实测频偏值
5. 如果实测值系统性偏大/偏小，可以微调代码中的 deviation 值

### Q2: 接收灵敏度低怎么办？

**A**: 逐步增加增益:
```c
// 第一步
(5u << 8)  // 增益 = 5

// 如果还是低
(6u << 8)  // 增益 = 6

// 最大不要超过
(7u << 8)  // 增益 = 7
```

同时检查天线和射频前端是否正常。

### Q3: 误码率高怎么办？

**A**: 可能的原因和解决方法:
1. **带宽太窄** → 增加带宽 (6→7→8)
2. **增益太高导致失真** → 降低增益 (5→4→3)
3. **频偏不准确** → 重新测量并调整 deviation
4. **晶振误差大** → 检查晶振或增加频率容差

### Q4: 完全没有信号怎么办？

**A**: 逐步排查:
1. 检查音频路径是否连通
2. 用示波器查看是否有音频输出
3. 检查 REG_30 是否正确配置
4. 确认 PTT 是否正常工作
5. 检查射频部分是否正常
