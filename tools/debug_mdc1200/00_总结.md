## 🎯 核心功能

### 1️⃣ 频偏测量 (最重要)
**作用**: 检查发射信号是否符合 MDC1200 标准

**标准值**: 
- 频偏：±3500 Hz
- Mark 频率：4700 Hz
- Space 频率：2300 Hz

**结果判断**:
- ✅ 优秀：误差 < 300 Hz
- ⚠️ 良好：300 Hz < 误差 < 800 Hz  
- ✗ 异常：误差 > 800 Hz

### 2️⃣ 频谱分析
**作用**: 查看信号的频率成分

**正常 MDC1200**:
- 4700 Hz 附近有峰值 (Mark)
- 2300 Hz 附近有峰值 (Space)

### 3️⃣ 波形显示
**作用**: 可视化信号特征

**三个视图**:
1. 时域波形 - 原始音频
2. 频谱图 - 频率分布
3. 语谱图 - 频率随时间变化

### 4️⃣ 自动报告
**作用**: 生成详细的分析报告

**包含内容**:
- 基本参数 (采样率、时长)
- 频率检测结果
- 频偏测量结果
- 调试建议
- 综合评分

## 🔧 固件修改要点

### 关键修改：频偏设置

**文件**: `App\driver\bk4829.c`  
**函数**: `BK4819_PlayMDC1200`  
**行号**: 约 1867 行

```c
// 修改前 (错误值)
uint16_t deviation = 850;

// 修改后 (正确值)
uint16_t deviation = 3500;
```

**为什么修改**:
- MDC1200 标准频偏是 ±3.5kHz
- 原代码 850Hz 远小于标准值
- 导致其他设备无法解调

### 辅助修改：接收优化

**文件**: `App\driver\bk4829.c`  
**函数**: `BK4819_EnableMDC1200Rx`  
**行号**: 约 1980 行

```c
// REG_58 配置
// 带宽：6 (原 7) - 更窄，提高信噪比
// 增益：5 (原 3) - 更高，提高灵敏度
```

## 📊 调试流程

### 发射调试流程

```
1. 修改频偏参数 (deviation = 3500)
   ↓
2. 编译并烧录固件
   ↓
3. 用调试工具录制发射信号
   ↓
4. 测量频偏
   ↓
5. 结果判断:
   - 误差 < 300Hz → ✅ 完成
   - 误差 > 300Hz → 微调 deviation 值
   ↓
6. 与其他设备互通测试
```

### 接收调试流程

```
1. 用标准设备发射
   ↓
2. 本机接收并录制
   ↓
3. 频谱分析
   ↓
4. 检查 Mark/Space 频率
   ↓
5. 结果判断:
   - 都有 → ✅ 接收正常
   - 缺失 → 调整接收参数
   ↓
6. 测试解码成功率
```

## 💡 使用技巧

### 技巧 1: 对比测试
用已知正常的设备发射，录制后作为参考标准。

### 技巧 2: 直接音频连接
```
对讲机耳机输出 → 音频线 → 电脑麦克风输入
```
避免空气传播的干扰和衰减。

### 技巧 3: 多次平均
每次测试录制 3-5 次，取平均值。

### 技巧 4: 记录日志
保存每次测试的报告截图，建立调试档案。

## 🐛 常见问题速查

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 无法录音 | 设备选择错误 | 刷新设备列表，选择正确设备 |
| 频偏误差大 | 参数设置错误 | 修改 deviation = 3500 |
| 检测不到频率 | 信号太弱 | 增加录音增益，靠近声源 |
| 只有一个峰值 | 不是 FSK | 检查发射端调制方式 |
| 打包失败 | 缺少 PyInstaller | pip install pyinstaller |
